Question 2
Which of the following is most suitable for being designated as Fourth Generation Language (4GL)? (UGC NET 3rd Dec 2019 Morning Paper)
  • A.ALGOL
  • B.Java
  • C.C
  • D.SQL
Correct Answer: SQL

SQL (Structured Query Language) is a Fourth Generation Language (4GL). 4GLs are a class of programming languages that are more abstract and user-friendly compared to lower-level languages like assembly and even traditional high-level languages like C or Java.

 

SQL is primarily used in the context of relational databases and is specifically designed for querying, manipulating, and managing data in a database. It allows users to interact with databases using natural language-like statements, making it easier to retrieve, update, and manage data. SQL statements are often close to human language and are highly specialized for database operations. SQL is widely used in various database management systems (DBMS), and it is a vital tool for data manipulation and reporting.

 

There are five generation of Programming languages:

1) First Generation Languages: These are low-level languages like machine language.

2) Second Generation Languages: These are low-level assembly languages used in kernels and hardware drives.

3) Third Generation Languages: These are high-level languages like C, C++, Java, Visual Basic and JavaScript.

4) Fourth Generation Languages: These are languages that consist of statements that are similar to statements in the human language. These are used mainly in database programming and scripting. Example of these languages include Perl, Python, Ruby, SQL, MatLab (MatrixLaboratory).

5) Fifth Generation Languages: These are the programming languages that have visual tools to develop a program. Examples of fifth generation language include Mercury, OPS5, and Prolog.

 

The first two generations are called low level languages. The next three generations are called high level languages.

Question 3

  • C.Machine codes are in shorthand English; mnemonic codes are in binary.
  • D.Machine codes are in shorthand English; mnemonic codes are a high-level language
Correct Answer: Machine codes are in binary; mnemonic codes are in shorthand English.

The primary difference between mnemonic codes and machine codes lies in their representation and human readability.

 

Machine codes are binary representations of instructions that a computer's CPU can directly execute. They consist of sequences of 0s and 1s and are not easily understandable by humans. In contrast, mnemonic codes are human-readable representations of machine instructions presented in a more understandable and symbolic form.

 

Mnemonics use short, descriptive terms that are easier for programmers to comprehend. They serve as an intermediate step between high-level programming languages and machine code, making it simpler for programmers to write and understand assembly language programs. Mnemonic codes improve the efficiency and clarity of programming for low-level operations.

Question 3
Which one of the following is not a Network Topology?
  • A.Bus
  • B.Chain
  • C.Ring
  • D.Star
Correct Answer: Chain

A network topology is the arrangement of a network, including its nodes and connecting lines. There are two ways of defining network geometry: the physical topology and the logical (or signal) topology.

 

The physical topology of a network is the actual geometric layout of workstations. There are several common physical topologies, as described below and as shown in the illustration.

 

In the bus network topology, every workstation is connected to a main cable called the bus. Therefore, in effect, each workstation is directly connected to every other workstation in the network.

 

In the star network topology, there is a central computer or server to which all the workstations are directly connected. Every workstation is indirectly connected to every other through the central computer.

 

In the ring network topology, the workstations are connected in a closed loop configuration. Adjacent pairs of workstations are directly connected. Other pairs of workstations are indirectly connected, the data passing through one or more intermediate nodes.

 

If a Token Ring protocol is used in a star or ring topology, the signal travels in only one direction, carried by a so-called token from node to node.

 

The mesh network topology employs either of two schemes, called full mesh and partial mesh. In the full mesh topology, each workstation is connected directly to each of the others. In the partial mesh topology, some workstations are connected to all the others, and some are connected only to those other nodes with which they exchange the most data.

 

The tree network topology uses two or more-star networks connected together. The central computers of the star networks are connected to a main bus. Thus, a tree network is a bus network of star networks.

Question 19
Which networking device connect one LAN to other LAN using same protocol?
  • A.Router
  • B.Switch
  • C.Bridge
  • D.Modem
Correct Answer: Bridge

A bridge is a networking device that connects one LAN (Local Area Network) to another LAN using the same protocol. The primary purpose of a bridge is to interconnect two separate LANs, and it does so by examining the MAC (Media Access Control) addresses of data packets and making forwarding decisions based on these addresses.

 

When a bridge receives data from one LAN, it inspects the MAC address of the source and destination devices in the data packet. If it determines that the destination device is on the other LAN, it forwards the data packet only to the LAN where the destination device resides. In this way, the bridge effectively connects the two LANs, allowing them to communicate with each other seamlessly.

 

Bridges are particularly useful in scenarios where different LAN segments need to be connected while maintaining a single network and using the same communication protocol. They operate at the data link layer (Layer 2) of the OSI model and play a crucial role in extending network coverage and improving network efficiency.
